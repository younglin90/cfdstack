# 논문 → 코드 변환 가이드

## 개요

논문의 수식을 코드로 옮기는 과정에서 발생하는 오류의 대부분은 **수식 자체의 이해 부족**이 아니라 **표기법·인덱스·부호·차원의 불일치**에서 온다. 이 스킬은 논문을 코드로 변환할 때 체계적으로 실수를 방지하는 프로토콜을 제공한다.

**핵심 원칙:** 코드를 한 줄이라도 작성하기 전에, 논문의 수식을 코드의 언어로 완전히 번역한다.

## 사전 조건

- 구현하려는 논문/교과서의 정확한 출처 (저자, 연도, 식 번호)
- 기존 솔버 코드의 인덱스 컨벤션, 좌표계, 변수 배열 구조 파악
- 기존 코드가 사용하는 단위계 (SI, CGS, 무차원) 확인

## 프로토콜

### Step 1: 수식 전사 (Transcription)

논문의 수식을 **코드와 무관하게** 마크다운으로 옮긴다.

**필수 기록 항목:**

```markdown
## [스킴 이름] ([저자], [학술지] [연도], Eq.([번호]))

### 지배 방정식 (governing equation)
- 보존형 / 비보존형: [명시]
- 변수 벡터 Q = [ρ, ρu, ρv, ρw, ρE]^T  (보존 변수)
- 원시 변수 W = [ρ, u, v, w, p]^T

### 이산화 수식
[논문의 식을 그대로 옮긴다. 논문 표기 그대로 사용.]

### 논문의 표기 규칙
- 아래첨자 i: [의미 — 셀 중심? 노드?]
- 아래첨자 i+1/2: [의미 — 셀 면? 중간점?]
- 위첨자 n: [의미 — 시간 단계?]
- 법선 벡터 방향: [owner→neighbor? 왼쪽→오른쪽?]
```

**절대 하지 말 것:**
- 논문 수식을 "이해했으니 생략"하고 바로 코드 작성
- 여러 논문의 표기를 혼합하여 사용
- 수식 전사 없이 "간단하니까 바로 구현"

### Step 2: 컨벤션 매핑 테이블

논문의 표기와 기존 코드의 표기를 1:1로 매핑하는 테이블을 작성한다.

```markdown
### 인덱스 매핑
| 논문 표기 | 의미 | 코드 표현 | 비고 |
|-----------|------|-----------|------|
| i | 셀 중심 | cells[i] 또는 i (0-based) | 논문은 1-based |
| i+1/2 | 셀 경계면 (오른쪽) | faces[i] 또는 i+1 | face 인덱스 정의 확인 |
| i-1/2 | 셀 경계면 (왼쪽) | faces[i-1] 또는 i | |
| L, R | face 기준 좌/우 상태 | left[f], right[f] | owner/neighbor 매핑 |

### 변수 매핑
| 논문 기호 | 의미 | 코드 변수명 | 단위 |
|-----------|------|-------------|------|
| ρ | 밀도 | rho | kg/m³ |
| u_i | 속도 i-성분 | vel[i] | m/s |
| p | 압력 | pres | Pa |
| a | 음속 | sos | m/s |
| H | 총 엔탈피 | H_total | J/kg |

### 부호/방향 매핑
| 항목 | 논문 정의 | 코드 정의 | 변환 필요 |
|------|-----------|-----------|-----------|
| 면 법선 벡터 | 왼쪽→오른쪽 | owner→neighbor | 확인 필요 |
| flux 부호 | 셀 밖으로 양 | [코드 확인] | |
```

**이 테이블 없이 코딩을 시작하지 않는다.**

### Step 3: 생략된 유도 복원

논문에서 자주 생략하는 단계를 명시적으로 복원한다.

**흔한 생략 패턴:**

| 논문 표현 | 실제 의미 | 해야 할 일 |
|-----------|-----------|------------|
| "trivially follows" | 3~10줄의 대수적 조작 | 중간 단계를 직접 유도 |
| "it can be shown that" | 비자명한 유도가 있음 | 유도를 수행하거나 참조 찾기 |
| "similarly for y, z" | x 방향만 보여주고 나머지 생략 | y, z를 명시적으로 전개 (좌표계 주의) |
| "after linearization" | Taylor 전개 + 고차항 버림 | 어떤 변수 기준 전개인지, 어디서 자른지 확인 |
| "using standard relations" | 열역학 관계식 사용 | 어떤 관계식인지 명시 (이상기체? 실제기체?) |
| "with appropriate limiting" | limiter 적용 | 어떤 limiter인지, minmod? van Leer? |
| "boundary terms vanish" | 특정 BC 가정 | 어떤 BC에서 사라지는지, 다른 BC에서는? |
| Eq.(A)에서 Eq.(B)로 점프 | 중간 변환 생략 | A와 B 사이의 단계를 직접 유도 |

**각 생략을 발견할 때마다 마크다운에 기록한다:**
```markdown
### 생략 복원 #1: Eq.(5) → Eq.(6) 사이
- 논문: "After substituting (3) into (4), we obtain (6)"
- 복원: [직접 대입하여 중간 단계 작성]
- 검증: 복원 결과가 Eq.(6)과 일치하는지 확인
```

### Step 4: 차원 검증 (Dimensional Analysis)

**모든 항의 차원이 일치하는지 검증한다.**

```markdown
### 차원 검증
| 수식 항 | 물리량 | 차원 | 일치 |
|---------|--------|------|------|
| ∂(ρu)/∂t | 운동량/시간/체적 | kg/(m²·s²) | ✓ |
| ∂(ρu² + p)/∂x | 운동량 flux의 발산 | kg/(m²·s²) | ✓ |
| τ_xx | 점성 응력 | Pa = kg/(m·s²) | ⚠ 차원 불일치 → 확인 |
```

**무차원 ↔ 유차원 변환 시:**
- 논문이 무차원이면: 기준값(reference) 명시 (ρ_ref, u_ref, L_ref, p_ref)
- 코드가 유차원이면: 모든 식에 기준값을 복원
- 코드가 무차원이면: 기준값으로 나누는 위치 명시
- **절대: 한 식 안에 무차원 항과 유차원 항을 혼용하지 않는다**

### Step 4.5: 수치 정밀도 함정 (Numerical Precision Traps)

코드에서는 수학적으로 동등한 식이 수치적으로 다른 결과를 줄 수 있다.

**흔한 정밀도 함정:**

| 문제 | 위험한 식 | 안전한 식 |
|------|-----------|-----------|
| 큰 수의 상쇄 | `p = (γ-1)·(ρE - 0.5·ρ·|v|²)` (고마하 시 ρE ≈ 0.5ρ|v|²) | 내부 에너지를 별도 변수로 추적 |
| 작은 수의 나눗셈 | `u = ρu / ρ` (ρ → 0 근처) | `u = ρu / max(ρ, ε)` 또는 양보존 보장 |
| 제곱근의 음수 | `a = √(γ·p/ρ)` (p < 0 발생 시) | p를 floor로 보호 후 계산 |
| 지수 함수 오버플로 | `exp(arg)` (arg > 700) | `min(arg, 500)` 클리핑 |
| 로그의 음수 | `log(ρ)` (ρ ≤ 0) | 양보존 스킴 사용 또는 floor |

**Kahan 보상 합산**: 셀이 매우 많을 때 보존량 합산에서 부동소수점 누적 오차가 문제가 될 수 있다. 보존 검증 시 주의.

### Step 5: 단위 함수별 구현

수식 전사와 매핑이 끝나면, **가장 작은 단위부터** 구현한다.

```
구현 순서:
1. 열역학 관계식 (EOS, 음속 등) — 독립적, 검증 쉬움
2. 재구성 (MUSCL, WENO 등) — 입출력이 명확
3. Flux 함수 — 재구성 결과를 입력받음
4. 시간 적분 — flux를 사용
5. 경계조건 — 전체 루프에 영향
```

**각 함수 구현 후 반드시:**
- 알려진 입력에 대한 기대 출력 확인 (손계산 또는 논문 Table)
- 극한 케이스 테스트 (M→0, M→∞, p_L=p_R 등)

### Step 6: 논문과의 1:1 대조

구현 완료 후, 코드의 각 줄이 논문의 어떤 식에 대응하는지 주석으로 표시한다.

```
// Eq.(18): face mass flux
m_half = a_half * (rho_L * M_plus + rho_R * M_minus);

// Eq.(20): split Mach number (subsonic branch, |M| <= 1)
M_plus  =  0.25 * (M_L + 1.0) * (M_L + 1.0);  
M_minus = -0.25 * (M_R - 1.0) * (M_R - 1.0);
```

**주석이 달리지 않는 코드 줄이 있으면 → 논문에 없는 로직을 추가한 것 → 근거를 명시하거나 제거.**

## 체크리스트

구현 완료 전 반드시 확인:

### 전사 완료
- [ ] 구현하는 모든 식의 논문 번호가 기록되어 있는가
- [ ] 논문에서 사용하는 모든 기호가 정의되어 있는가
- [ ] 생략된 유도가 모두 복원되었는가

### 컨벤션 일치
- [ ] 인덱스 매핑 테이블이 작성되었는가 (논문 ↔ 코드)
- [ ] 변수 매핑 테이블이 작성되었는가
- [ ] 부호/방향 매핑이 확인되었는가
- [ ] 논문이 1-based인데 코드가 0-based이면 → 오프셋 적용 위치가 명시되었는가

### 차원 일치
- [ ] 모든 항의 차원이 검증되었는가
- [ ] 무차원/유차원 혼용이 없는가
- [ ] 기준값(reference values)이 명시되었는가 (무차원 사용 시)

### 코드 대조
- [ ] 코드의 각 계산 줄에 논문 식 번호 주석이 있는가
- [ ] 논문에 없는 로직에 근거 주석이 있는가
- [ ] 극한 케이스 테스트를 통과하는가

## Red Flags — STOP

| 상황 | 조치 |
|------|------|
| 수식 전사 없이 코딩 시작 | 중단. Step 1로 돌아간다. |
| 인덱스 매핑 테이블 없이 루프 작성 | 중단. Step 2로 돌아간다. |
| "trivially follows"를 복원하지 않고 넘어감 | 중단. Step 3로 돌아간다. |
| 차원 검증 없이 "수식이 맞으니까" | 중단. Step 4로 돌아간다. |
| 여러 논문의 표기를 한 함수에서 혼용 | 중단. 하나의 논문 표기로 통일한다. |
| 코드 줄에 대응하는 논문 식을 못 찾음 | 중단. 근거 없는 코드는 제거하거나 근거를 명시한다. |
| "x 방향만 구현하고 y, z는 나중에" | 중단. y, z도 수식을 먼저 전개한다 (좌표계 차이 가능). |

## 흔한 실수 카탈로그

### 1. 부호 오류 (Sign Errors)
- **면 법선 방향**: 논문은 L→R이 양인데, OpenFOAM의 `Sf()`는 owner→neighbour. 뒤집으면 모든 flux가 반전됨
- **flux 부호**: 보존 법칙의 발산 정리 적용 시 면적분의 부호
- **압력항**: momentum flux에서 p·n 항의 방향
- **점성 응력 부호**: 일부 논문은 τ에 음의 부호를 포함 (압력 부호 규약). 혼용하면 점성 기여가 2배 또는 0이 됨
- **소스항**: 음의 소스(sink)를 양의 부호로 쓰는 논문이 있음

### 2. 인덱스 오프바이원 (Off-by-One)
- **면 인덱스**: 논문의 face i+1/2가 코드의 face[i]인지 face[i+1]인지 — WENO/MUSCL 버그의 가장 흔한 원인
- **스텐실 오프셋**: WENO-5 서브스텐실 S0,S1,S2의 순서(left-biased→right-biased)가 뒤집히면, smooth 영역에서는 정상이나 충격파에서 ENO 선택이 반전됨
- **ghost cell 개수**: WENO-5는 한 쪽에 3개 ghost cell 필요. MUSCL용 2개만 있으면 초기화되지 않은 메모리를 읽음 (조밀 격자에서만 증상 발현)
- **반복문 범위**: `for i in 0..N` vs `for i in 1..N-1` (경계 제외)
- **cell-center vs face-center 저장**: 논문은 값이 셀 중심에 있다고 가정하지만, 코드는 flux를 face에 저장. 이 shift를 빠뜨리면 스텐실이 잘못된 위치를 참조

### 3. 무차원화 불일치
- **기준값 누락**: 논문의 Re, Ma를 사용하는데 기준 속도/길이가 다름
- **압력 스케일링**: 논문이 `p* = p / (ρ_inf * U_inf²)`로 무차원화했는데, 코드에서 유차원 압력을 사용하면 Re 스케일의 오차 발생
- **에너지 방정식**: 무차원 상태방정식 `p = ρT / (γ·M_inf²)`에서 `M_inf²`를 빠뜨리는 것이 전형적 실수
- **점성 계수**: μ를 Re로 대체할 때 기준값 위치

### 4. 좌표계 불일치
- **곡선 좌표계**: 논문이 Cartesian인데 코드가 curvilinear → Jacobian 스케일링 누락 시 freestream preservation 위반 (GCL 조건 위반, SU2 issue #572 참조)
- **원통 좌표계**: 1/r 기하학 소스항을 빠뜨림
- **2D 가정**: 논문의 2D 식을 3D 코드에 넣을 때 z-방향 처리

### 5. "Similarly" 함정
- **y-방향**: 논문이 "x와 동일"이라고 했지만 곡선좌표에서는 metric이 다름
- **에너지**: "운동량과 유사"라고 했지만 압력 일 항이 추가됨
- **점성항**: "x-운동량과 동일"이라고 했지만 교차 미분항이 다름

### 6. 생략된 핵심 디테일
- **Roe 평균**: Roe(1981)는 parameter vector `z = √ρ · [1, u, H]`를 제시하지만 엔탈피 평균의 유도를 생략. 산술 평균으로 대체하면 Rankine-Hugoniot 조건을 깨뜨림
- **엔트로피 보정**: 많은 논문이 Roe flux를 제시한 후 entropy fix를 한 문장으로 언급. 이를 빠뜨리면 팽창 충격파(비물리적) 발생
- **벽함수 y+ 범위**: 논문은 30 < y+ < 300에서 유도하지만, 코드가 y+ < 5에서도 적용하면 벽면 전단응력이 크게 왜곡됨

### 7. 경계조건 구현 갭
- **아음속 입구/출구**: 논문의 "non-reflecting BC"는 특성 분해 기반인데, 구현 시 모든 변수를 고정(over-specified)하면 반사파 발생
- **대칭 BC**: "법선 속도 반사"라고 했지만 운동량의 법선 성분과 에너지도 함께 조정해야 함

## 관련 스킬
