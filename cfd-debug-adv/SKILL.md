# CFD 솔버 디버깅 심화

## 개요

솔버가 발산하거나 비물리적 해를 만들 때, "CFL을 낮춰보자"는 응급처치지 근본 해결이 아니다. 발산의 **패턴**을 먼저 분류하고, 패턴에 맞는 진단 경로를 따른다.

**핵심 원칙:** 발산 패턴을 분류하기 전에 수정을 시도하지 않는다. 한 번에 하나만 변경한다.

## 사전 조건

- 발산/오류가 재현 가능해야 한다
- 잔차(residual) 이력을 출력할 수 있어야 한다
- 필드 데이터를 중간 시간 단계에서 덤프할 수 있어야 한다

## 프로토콜

### Step 1: 발산 패턴 분류

먼저 증상을 관찰하고 아래 5가지 패턴 중 하나로 분류한다.

```
발산 유형은?
├── A. 즉시 발산 (1~5 스텝 내 NaN/Inf)
├── B. 점진적 발산 (수십~수백 스텝에 걸쳐 값이 커짐)
├── C. 특정 시간/위치에서 발산 (일정 시간 정상 진행 후 갑자기)
├── D. 잔차 정체 (발산은 아니지만 수렴하지 않음)
└── E. 비물리적 해 (발산은 안 하지만 결과가 틀림)
```

### Step 2: 패턴별 진단

#### 패턴 A: 즉시 발산

**가능한 원인 (우선순위 순):**
1. **초기조건 오류**: ρ ≤ 0, p ≤ 0, T ≤ 0이 초기 필드에 존재
2. **경계조건 오류**: ghost cell이 초기화되지 않음, BC 값이 비물리적
3. **flux 부호 오류**: 면 법선 방향이 반대 → 첫 스텝부터 발산
4. **dt 계산 오류**: CFL 기반 dt가 극단적으로 큼 (체적/면적 계산 오류)
5. **영 나눗셈**: 밀도가 0인 셀에서 속도 = 운동량/밀도 계산

**진단 절차:**
```
1. 첫 번째 시간 단계 전의 모든 필드를 출력한다 (초기조건 + ghost cell)
2. min/max를 확인한다: ρ_min, p_min, T_min > 0?
3. dt 값을 확인한다: 예상 범위 내인가?
4. 첫 번째 시간 단계 후의 필드를 출력한다
5. NaN/Inf가 처음 나타나는 셀 위치를 찾는다 → 그 셀의 face flux를 역추적
```

#### 패턴 B: 점진적 발산

**가능한 원인:**
1. **보존 위반**: flux의 보존성이 깨져서 에너지가 누적
2. **불안정한 시간 적분**: explicit 스킴에서 CFL > 1
3. **경계 반사**: non-reflecting BC가 제대로 작동하지 않아 에너지 축적
4. **소스항 불균형**: 생성 > 소멸이 지속됨

**진단 절차:**
```
1. 전체 보존량 시간 이력을 출력한다: Σ(ρ·V), Σ(ρu·V), Σ(ρE·V)
2. 보존량이 단조 증가/감소하는가? → 보존 위반
3. CFL 수를 시간에 따라 출력한다 → 1을 초과하는 시점이 있는가?
4. 최대값이 증가하는 셀의 위치를 추적한다 → 경계 근처인가, 내부인가?
5. CFL을 절반으로 줄여서 재실행한다 → 발산 시점이 2배로 늘어나면 CFL 문제
```

#### 패턴 C: 특정 시간/위치에서 발산

**가능한 원인:**
1. **충격파 도달**: limiter가 없거나 부족한 위치에 충격파 도달
2. **와류 붕괴**: under-resolved vortex가 비물리적 음압을 생성
3. **격자 특이점**: 격자 품질이 나쁜 특정 셀 (높은 skewness, 극단적 종횡비)
4. **경계조건 전환**: 시간 의존 BC의 불연속적 변화
5. **물성치 범위 초과**: 온도 의존 점성계수가 극단값에서 발산

**진단 절차:**
```
1. 발산 직전 시간 단계의 필드를 덤프한다
2. NaN/극단값이 처음 나타나는 셀을 찾는다
3. 그 셀의 특성을 확인한다: 격자 품질, 주변 셀과의 값 차이, 경계와의 거리
4. 그 셀의 시간 이력을 출력한다: 어느 변수가 먼저 발산하는가?
5. 해당 위치의 물리 현상을 파악한다: 충격파? 와류? 분리?
```

#### 패턴 D: 잔차 정체

**가능한 원인:**
1. **선형 솔버 수렴 실패**: GMRES/BiCGSTAB가 내부 반복 한계 도달
2. **시간 단계가 너무 큼**: implicit이라도 정확도가 떨어져 잔차가 줄지 않음
3. **limiter 진동**: limiter가 두 상태 사이에서 flip-flop
4. **경계조건 불일치**: 정상 상태 해가 존재하지 않는 BC 조합

**진단 절차:**
```
1. 선형 솔버의 내부 잔차를 출력한다 → 수렴하는가?
2. 잔차가 가장 큰 셀의 위치를 찾는다 → 특정 영역에 집중되어 있는가?
3. dt를 1/10로 줄여서 재실행한다 → 잔차가 더 줄어드는가?
4. 1차 스킴으로 전환한다 → 수렴하는가? (limiter 문제 판별)
```

#### 패턴 E: 비물리적 해

**증상:**
- 음의 밀도/압력 (ρ < 0, p < 0)
- 비물리적으로 높은 온도/마하수
- 대칭 문제에서 비대칭 해
- 보존량이 시간에 따라 변함 (정상 상태에서)
- 수렴 차수가 이론보다 낮음

**진단 절차:**
```
1. 어떤 물리량이 비물리적인지 특정한다
2. 비물리적 값이 나타나는 셀 위치를 매핑한다
3. /cfd-vv를 사용하여 수렴 차수를 확인한다 → 이론과 불일치하면 구현 오류
4. 1D로 단순화하여 동일 증상이 재현되는지 확인한다
5. 해석해가 있는 문제에서 동일 증상이 나타나는지 확인한다
```

### Step 3: NaN 전파 추적

NaN이 발생한 경우, 최초 발생 지점을 정확히 찾는다.

**절차:**

1. **NaN 감지 삽입**: 모든 필드 업데이트 후 NaN 검사 코드를 삽입한다
   ```
   각 셀 i에 대해:
     if isnan(Q[i]) or isinf(Q[i]):
       출력: "NaN at cell i=[좌표], step n=[시간단계], variable=[변수]"
       직전 시간 단계의 해당 셀 + 이웃 셀 값 출력
       중단
   ```

2. **역추적**: NaN이 처음 나타난 셀에서 역방향으로 추적한다
   ```
   NaN 셀의 face flux 확인 → 어떤 face에서 NaN이 들어왔는가?
   → 그 face의 좌/우 상태 확인 → 재구성 값인가? 원본 값인가?
   → 재구성이면 → stencil 내의 어떤 셀에서 비물리적 값이 시작되었는가?
   ```

3. **원인 특정**: NaN의 산술적 원인을 분류한다
   | NaN 유형 | 산술 원인 | 흔한 CFD 원인 |
   |----------|-----------|---------------|
   | 0/0 | 0으로 나눔 | ρ=0인 셀에서 u=ρu/ρ |
   | ∞-∞ | 큰 수의 상쇄 | 보존 변수→원시 변수 변환 |
   | sqrt(음수) | 음수 제곱근 | p<0에서 음속 a=√(γp/ρ) |
   | log(음수) | 음수 로그 | 온도<0에서 점성계수 계산 |

### Step 4: Jacobian 검증

Implicit 솔버에서 수렴 실패 시, Jacobian의 정확성을 검증한다.

**해석적 Jacobian vs 수치적 Jacobian 비교:**

```
수치 Jacobian의 (i,j) 성분:
  J_num[i][j] = (R(Q + ε·e_j) - R(Q - ε·e_j)) / (2ε)

여기서:
  R = 잔차 함수
  e_j = j번째 단위 벡터
  ε = sqrt(machine_eps) * max(|Q[j]|, 1)

비교:
  relative_error[i][j] = |J_analytic[i][j] - J_num[i][j]| / max(|J_num[i][j]|, 1e-30)

판정:
  relative_error < 1e-4  →  PASS
  1e-4 < relative_error < 1e-2  →  경고 (수치 미분 정밀도 문제일 수 있음)
  relative_error > 1e-2  →  FAIL (Jacobian 구현 오류)
```

**주의:**
- 모든 셀이 아니라 몇 개의 대표 셀에서만 검증한다 (경계 셀, 내부 셀, 고 gradient 셀)
- flux Jacobian과 소스항 Jacobian을 분리하여 검증한다
- 경계 조건의 Jacobian도 별도 검증한다

### Step 5: 차원 축소 디버깅

위 단계에서 원인을 특정하지 못했으면, 문제를 체계적으로 단순화한다.

```
단순화 경로:

3D 문제
├── 2D로 축소 (z-방향 제거)
│   ├── 재현됨 → 2D에서 디버깅 계속
│   └── 재현 안 됨 → 3D 특유 문제 (metric term, 교차 미분항)
│
├── 1D로 축소
│   ├── 재현됨 → 1D에서 디버깅 (가장 빠름)
│   └── 재현 안 됨 → 다차원 상호작용 문제
│
├── 공간 차수 하강 (high-order → 1st order)
│   ├── 1st order에서 수렴 → 재구성/limiter 문제
│   └── 1st order에서도 발산 → flux 함수 또는 시간 적분 문제
│
├── 시간 적분 변경 (implicit → explicit Euler, CFL << 1)
│   ├── explicit에서 수렴 → implicit Jacobian 또는 선형 솔버 문제
│   └── explicit에서도 발산 → 공간 이산화 문제
│
└── 경계 제거 (주기 경계만 사용)
    ├── 주기 경계에서 수렴 → BC 구현 문제
    └── 주기 경계에서도 발산 → 내부 스킴 문제
```

**각 단순화에서 반드시 한 가지만 변경한다.** 동시에 차수와 격자를 바꾸면 원인 특정이 불가능하다.

### Step 6: 3회 수정 실패 시

3번의 수정 시도가 모두 실패하면:

1. **수식 재유도**: `/cfd-paper-to-code`의 Step 1~3을 처음부터 다시 수행한다
2. **참조 구현 비교**: 동일 스킴의 오픈소스 구현(OpenFOAM, SU2 등)과 줄 단위 비교
3. **사용자에게 보고**: 현재까지의 진단 결과를 정리하여 AskUserQuestion으로 방향 확인

## 체크리스트

### 디버깅 시작 전
- [ ] 발산이 재현 가능한가
- [ ] 잔차 이력을 출력할 수 있는가
- [ ] 발산 패턴을 A~E 중 하나로 분류했는가

### 진단 중
- [ ] 한 번에 하나만 변경하고 있는가
- [ ] 변경 전후의 결과를 기록하고 있는가
- [ ] NaN의 최초 발생 위치를 특정했는가

### 수정 후
- [ ] 원래 발산하던 케이스에서 더 이상 발산하지 않는가
- [ ] 수정으로 인해 다른 벤치마크가 깨지지 않았는가
- [ ] 수렴 차수가 유지되는가 (`/cfd-vv`로 확인)

## Red Flags — STOP

| 상황 | 조치 |
|------|------|
| 발산 패턴 분류 없이 "CFL을 줄여보자" | 중단. Step 1로. |
| CFL, flux, BC를 동시에 수정 | 중단. 하나만. |
| NaN 위치를 찾지 않고 코드 전체를 리뷰 | 중단. Step 3으로. |
| "잘 모르겠으니 limiter를 추가하자" | 중단. 원인을 먼저 특정. |
| implicit 솔버 발산 시 Jacobian 검증 없이 dt만 줄임 | 중단. Step 4로. |
| 3회 수정 실패 후 4번째 시도 | 중단. Step 6으로 (수식 재유도). |

## 관련 스킬

- `/cfd-paper-to-code` — 수식 재유도 시 활용
- `/cfd-vv` — 수정 후 수렴 차수 재확인
- `/cfd-flux` — flux 함수 관련 발산 시 참조
- `/cfd-implicit` — Jacobian/선형 솔버 문제 시 참조

- `systematic-debugging` — 일반적 root cause 추적 프레임워크
